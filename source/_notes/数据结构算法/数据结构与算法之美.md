## 大 O 时间复杂度

> T(n) = O(f(n))

- T(n)：代码执行时间
- n：数据规模大小
- f(n) ：每行代码执行次数总和
- O：代码执行时间 T(n) 与 f(n) 表达式成正比

**表示**：代码执行时间随数据规模增长的**变化趋势**

**可忽略**：低阶、常量、系数不左右增长趋势的三部分

```
T(n)=O(2n+2) => T(n)=O(n)
T(n)=O(2n^2+2n+3) => T(n)=O(n2)
```

## 时间复杂度分析

### 诀窍

- 只关注循环次数最多的一次代码
- 加法法则：总复杂度等于量级最大的那段代码的复杂度
- 乘法法则：嵌套代码的负责度等于嵌套内外代码复杂度的乘积

```c
while (i <= n) 
{ 
	i = i * 2; 
}
```
