layer上下文

  • -drawRect:方法
    • 只有在这个方法里面才能获取到跟view的layer相关联的图形上下文［仅限制为layer上下文］
      • 重新绘制注意点
        • 不能手动调用，因为手动点用时不能获取图形上下文，我们自己创建不了，只能系统帮我们创建，并且传递给我们
        • 要想重新绘制，需要调用setNeedsDisplay
          • 这个方法并不会马上调用drawRect，其实这个方法只是给当前控件添加刷新的标记，等下一次屏幕刷新的时候才会调用drawRect，所以多次调用setNeedsDisplay并不会造成性能损失
    • 调用时间
      • 当这个view要显示的时候才会调用drawRect绘制图形
        • 在viewWillAppear之后在viewDidAppear之前调用
      • 使用setNeedsDisplay后，在屏幕进行刷新时也会进行调用
    • rect参数
      • 这个参数传进来的是控件的bounds
    • 注意，只要调用了这个方法，就会把前面绘制的东西全部清除
    • 当渲染系统准备好，它会调用视图图层的-display方法，此时，图层会装配他的后备存储。然后建立一个CoreGraphics上下文(CGContextRef)，将后备存储对应内存中的数据恢复出来，绘图会进入对应的内存区域，并使用CGContextRef绘制。
    • 当使用UIKit的绘制方法如(UIRectFill()/[UIBezierPath fill])代替-drawRect方法，他们将会使用这个上下文。使用步骤是：UiKit将后备存储的CGContextRef推进他的上下文堆栈(貌似不管怎样，都会先push进管理的上下文堆栈)，也就是说，他会将那个上下文设置为当前的。因此UIGraphicsGetCurrent将会返回那个对应的上下文。如果想直接使用CoreGraphics方法，可以自己调用UIGraphicsGetCurrent得到相同的上下文，并且讲这个上下文传给CoreGraphics方法。
  • 图形上下文
    • 可以理解为画布，我的理解就是缓存，专门存放我要画到屏幕上的内容（因为CPU输送数据的速度总是快于屏幕显示的速度的，所以需要这么一个缓存）
    • 图形上下文必须跟view相关联，才能将内容绘制到view上面
    • 目前我们所用的上下文都是以UIGraphics开头进行获取
    • CGContextRef
      • Ref:引用
      • CG:目前使用到的类型和函数一般都是CG开头
  • 绘制图形的四个步骤
    • 1.获取上下文
      • UIGraphicsGetCurrentContext()这个函数用来获取各种上下文（bitmap、layer等）
    • 2.创建、绘制路径
    • 3.添加路径
    • 4.渲染
      • 有些绘制方式是使用系统默认创建的路径，或者调用了一集成的方法（如UIBezierPath的fill，就集成了1、3、4步骤），所以，只要是绘图，一定会经历这4个步骤

  • 绘制饼状图和柱状图
    • 绘制饼状图
    - (void)drawRect:(CGRect)rect
{
  // 获取随机数值数组
  NSArray *array = [self randomIntegerArrayWithin:100 elementNumber:4];

  CGPoint center = CGPointMake(rect.size.width * 0.5, rect.size.height * 0.5);
  CGFloat startAngle = 0;
  CGFloat endAngle = 0;

  for (NSNumber *number in array) {
      endAngle = startAngle + [number floatValue] / 100.0 * M_PI * 2;

      UIBezierPath *path = [UIBezierPath bezierPathWithArcCenter:center radius:rect.size.width * 0.5 startAngle:startAngle endAngle:endAngle clockwise:YES];
      [path addLineToPoint:center];

      // 获取随机颜色
      [[self randomColor] set];
      [path fill];

      // 下一个弧形的起始角度等于当前弧形的结束角度
      startAngle = endAngle;
  }
}
    • 绘制柱状图
    - (void)drawRect:(CGRect)rect
{
  NSInteger count = 4;
  NSInteger max  = 100;

  // 获取随机数值数组
  NSArray *array = [self randomIntegerArrayWithin:max elementNumber:count];

  CGFloat w = rect.size.width / (array.count * 2 - 1);
  __block CGFloat x = 0;
  __block CGFloat y = 0;
  __block CGFloat h = 0;

  [array enumerateObjectsUsingBlock:^(id obj, NSUInteger idx, BOOL *stop) {
      x = w * idx * 2;
      h = [obj floatValue] / (CGFloat)max * rect.size.height;

      // 倒立着画
      y = rect.size.height - h;

      UIBezierPath *path = [UIBezierPath bezierPathWithRect:CGRectMake(x, y, w, h)];

      // 获取随机颜色
      [[self randomColor] set];
      [path fill];
  }];
}
  • 绘制动态图形
    • 绘制动态图形时，一般不使用NSTimer定时器，一般使用CADisplayLink
      • CADisplayLink无法设置调用时间，默认屏幕刷新时就会调用
      • 因为CADisplayLink的回调函数在屏幕刷新时就会调用，而-drawRect:在调用setNeedsDisplay后，刷新屏幕时也会调用，所以在CADisplayLink的回调函数中调用setNeedsDisplay，就能实现很顺畅的动画效果
     CADisplayLink *link = [CADisplayLink displayLinkWithTarget:self selector:@selector(timeChange)];
 // 要加进主循环中
  [link addToRunLoop:[NSRunLoop mainRunLoop] forMode:NSDefaultRunLoopMode];
  • 图形上下文状态栈
    • UIKit维护着一个上下文堆栈
    • 在渲染时，会获取图形上下文栈中的属性，对路径进行绘制，并且图形上下文状态栈不会自动清空，也就是下次添加路径并进行绘制时，使用的还是同一个状态栈，相同的状态，所以需要进行覆盖
    • 可以通过以下函数保存和还原状态栈的内容，从而实现后面添加的路径也能以保存状态前的属性进行绘制的功能
    // 保存当前的状态栈
CGContextSaveGState(ctx);

// 复原以前最后一个保存的状态栈
CGContextRestoreGState(ctx);
    • UIBezierPath类的一些改动
      • 如以下代码，不需要进行保存还原操作，第二条路径还是以原始的属性进行绘制
        • 之所以会这样，是因为Bezier把对路径的部分属性私有化了，并没有在状态栈中共享
      // 图形上下文对应的同一个，但是Bezier把对路径的部分属性私有化了
UIBezierPath *path = [UIBezierPath bezierPath];

[path moveToPoint:CGPointZero];
[path addLineToPoint:CGPointMake(100, 100)];
path.lineWidth = 20;
path.lineCapStyle = kCGLineCapRound;
[path stroke];

path = [UIBezierPath bezierPath];

[path moveToPoint:CGPointMake(100, 50)];
[path addLineToPoint:CGPointMake(200, 50)];

[path stroke];
      • 假如在以上代码添加颜色属性，如下所示
        • 第二条路径绘制出来还是红色，因为通过[[UIColor redColor] set]设置的属性在上下文状态栈中共享，并没有被私有化。所以还是需要通过存储状态栈的状态来实现用原始属性对第二条路径进行绘制。
      // 获取上下文
// CGContextRef ctx = UIGraphicsGetCurrentContext();

// 没有存储和复原，第二条路径绘制出来还是红色
//    CGContextSaveGState(ctx);

// 图形上下文对应的同一个，但是被色儿把对路径的部分属性私有化了
UIBezierPath *path = [UIBezierPath bezierPath];

[path moveToPoint:CGPointZero];

[path addLineToPoint:CGPointMake(100, 100)];
path.lineWidth = 20;
path.lineCapStyle = kCGLineCapRound;

// 颜色属性是在上下文状态栈中共享的
[[UIColor redColor] set];
[path stroke];

//    CGContextRestoreGState(ctx);
path = [UIBezierPath bezierPath];

[path moveToPoint:CGPointMake(100, 50)];
[path addLineToPoint:CGPointMake(200, 50)];

[path stroke];
      • Bezier的底层做法还是离不开绘图的4个步骤，通过以下代码得知，Bezier是在调用fill、stroke时获取上下文并将路径添加进上下文的
        • 上下文在进入drawRect:时系统就传入了，我们只需要获取就行，类似[[UIColor redColor] set]会默认设置到上下文中。
        • 所以可以在drawRect:外部创建Bezier路径，在drawRect:里面调用fill、stroke进行渲染
        • 输出
      // Bezier添加路径的上下文就是以下代码获取的上下文
CGContextRef ctx = UIGraphicsGetCurrentContext();

UIBezierPath *path = [UIBezierPath bezierPath];

[path moveToPoint:CGPointZero];

[path addLineToPoint:CGPointMake(100, 100)];

// 在stroke之前，从上下文获取的路径当前点不存在，说明是在stroke时获取上下文并将路径添加进上下文
NSLog(@"%@", NSStringFromCGPoint(CGContextGetPathCurrentPoint(ctx)));
[path stroke];

// 执行渲染操作后，上下文中的路径会被清空
NSLog(@"%@", NSStringFromCGPoint(CGContextGetPathCurrentPoint(ctx)));
      • 所以在进行矩阵操作时，Bezier可以直接这样写
        • 矩阵操作必须在添加路径之前操作
        - (void)drawRect:(CGRect)rect
{
CGContextRef ctx = UIGraphicsGetCurrentContext();

UIBezierPath *path = [UIBezierPath bezierPathWithOvalInRect:CGRectMake(0, 0, 300, 200)];

CGContextScaleCTM(ctx, 0.5, 0.5);
CGContextRotateCTM(ctx, M_PI_4);

// 以下方法将路径添加进上下文，并且渲染
[path fill];
}
      • 而手动添加路径的原生的绘制需要在addPath之前写
      - (void)drawRect:(CGRect)rect
{
CGContextRef ctx = UIGraphicsGetCurrentContext();

CGMutablePathRef path = CGPathCreateMutable();

CGPathMoveToPoint(path, NULL, 0, 0);
CGPathAddLineToPoint(path, NULL, 200, 200);

// 需要在addPath之前写
CGContextRotateCTM(ctx, M_PI_4);

// 向图形上下文添加路径
CGContextAddPath(ctx, path);

CGContextStrokePath(ctx);
}
      • 因为需要在添加路径前进行矩阵操作，所以使用系统默认创建的路径就不行了，矩阵操作不能生效
        • 按理论，路径被加进上下文，是被拷贝内容过去的，也就是说，一旦添加路径后，即使我再修改原先添加的路径也不能生效的，需要再进行一次添加。但是以下代码却能对已经在上下文中的路径进行修改，可能CGContext开头的函数就是操作的上下文中的默认路径
      - (void)drawRect:(CGRect)rect
{
CGContextRef ctx = UIGraphicsGetCurrentContext();

// 应该在这一句就在上下文中创建了默认路径
CGContextMoveToPoint(ctx, 20, 20);

// 输出有值，说明默认路径在创建时就被加入了上下文中
NSLog(@"%@", NSStringFromCGPoint(CGContextGetPathCurrentPoint(ctx)));

CGContextAddLineToPoint(ctx, 200, 200);

// 系统默认的路径已经添加进默认上下文了，所以以下矩阵操作无效
CGContextRotateCTM(ctx, M_PI_4);

CGContextStrokePath(ctx);
}
  • UIBezierPath
    • 使用UIBezierPath为什么不传入一个上下文参数到方法中？因为当使用UIKit或者AppKit时，上下文是唯一的。UIKit维护着一个上下文堆栈，UIKit方法总是绘制到最顶层的上下文中。
    // 获取顶层上下文
UIGraphicsGetCurrentContext()
// 在UiKit的堆栈中推进上下文
UIGraphicsPushContext(<#CGContextRef context#>)
// 在Uikit的堆栈中取出上下文
UIGraphicsPopContext()
